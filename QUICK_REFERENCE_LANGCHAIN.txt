# 🧠 LangChain Brain Features - Quick Reference

## Enable/Disable

```python
# Enable LangChain (requires: pip install langchain langchain-community langchain-huggingface chromadb sentence-transformers)
planner = PlannerAgent(use_langchain=True)   # Memory + Vector Store
generator = GeneratorAgent(use_langchain=True)  # Memory
healer = HealerAgent(use_langchain=True)     # Memory + Vector Store + RAG

# Disable (even if installed)
planner = PlannerAgent(use_langchain=False)
```

## PlannerAgent 🎯

### What It Does
- **Stores**: All test plans in vector database
- **Searches**: For similar past plans before creating new ones
- **Suggests**: Proven test scenarios from history
- **Remembers**: Planning context across sessions

### Methods
```python
# Automatic - stores plan after creation
plan = await planner.execute(target, config)

# Manual search
similar = await planner.search_similar_plans("login tests", k=3)
scenarios = await planner.retrieve_similar_scenarios(target, requirements)
```

### Result
```python
plan['langchain_insights'] = {
    "similar_plans_found": 2,
    "similar_plans": [...],
    "vectordb_enabled": True,
    "memory_enabled": True
}
```

## GeneratorAgent 🔧

### What It Does
- **Remembers**: Code generation patterns
- **Maintains**: Context across multiple generations
- **Learns**: Preferred coding styles

### Usage
```python
# Memory automatic - just use normally
generator = GeneratorAgent(use_langchain=True)
code = await generator.execute(target, config)
```

## HealerAgent 🩹 (Most Powerful!)

### What It Does
- **Stores**: All successful fixes in RAG database
- **Searches**: For similar past failures
- **Suggests**: Fixes that worked before (prioritized)
- **Learns**: From every successful healing
- **Gets Smarter**: Over time with accumulated knowledge

### Methods
```python
# Automatic - stores + searches during healing
healing = await healer.execute(test_id, config)

# Manual search
similar_failures = await healer.search_similar_failures(
    error_message="Timeout",
    failure_type="timeout",
    k=5
)

# Get RAG suggestions
suggestions = await healer.get_rag_based_fix_suggestions(analysis)
```

### Result
```python
healing['langchain_insights'] = {
    "similar_failures_found": 3,
    "rag_suggestions": 2,
    "past_fixes_referenced": [...],
    "vectordb_enabled": True,
    "memory_enabled": True
}

healing['recommended_fixes'][0] = {
    "type": "rag_based_fix",
    "description": "Similar timeout fixed before",
    "confidence": 92,
    "source": "RAG - Similar past fix",
    "details": "..."
}
```

## Vector Stores

### Locations
```
playwright_plans_db/     # PlannerAgent
playwright_healing_db/   # HealerAgent
```

### Technology
- **Database**: Chroma (SQLite-based)
- **Embeddings**: HuggingFace sentence-transformers/all-MiniLM-L6-v2
- **Storage**: Persistent (saved to disk)

### Custom Paths
```python
planner = PlannerAgent(
    use_langchain=True,
    persist_dir="./my_custom_db"
)
```

## Memory

### Type
- ConversationBufferMemory from LangChain
- Tracks context within agent instance

### Access
```python
if planner.memory:
    history = planner.memory.load_memory_variables({})
```

## RAG (Retrieval Augmented Generation)

### How It Works
```
1. New failure detected
2. Convert to embeddings
3. Search vector store (semantic similarity)
4. Retrieve successful past fixes
5. Rank by similarity score
6. Suggest top fixes first in recommended_fixes[]
```

### Similarity Thresholds
- **High**: > 0.85 (Very similar - high confidence)
- **Medium**: 0.70 - 0.85 (Similar - good confidence)
- **Low**: < 0.70 (Somewhat similar - use with caution)

## Common Patterns

### Create Plan with RAG
```python
planner = PlannerAgent(use_langchain=True)

# First plan
plan1 = await planner.execute("Login", {...})
# similar_plans_found: 0

# Second similar plan
plan2 = await planner.execute("Authentication", {...})
# similar_plans_found: 1 (found plan1!)
```

### Heal with RAG Learning
```python
healer = HealerAgent(use_langchain=True)

# Fix a failure
healing1 = await healer.execute("test_001", {
    "error_info": {"message": "Timeout", "type": "TimeoutError"},
    "auto_fix": True  # Important: stores fix in RAG
})

# Fix similar failure later
healing2 = await healer.execute("test_002", {
    "error_info": {"message": "Timeout waiting...", "type": "TimeoutError"}
})
# RAG suggests fix from healing1!
```

## Performance

### First Use
- Vector stores empty
- No similar items found
- Agents work normally

### After 10 Uses
- Starting to find similarities
- 1-2 suggestions per operation

### After 100 Uses
- Rich knowledge base
- 5-10 suggestions per operation
- Significant accuracy improvement

- Expert-level suggestions

| **Generation** | Generates code | Generates + Remembers |

## Troubleshooting

### "No module named 'langchain'"
```bash
pip install langchain langchain-community langchain-huggingface chromadb sentence-transformers
```

### "Vector store not found"
- First time: Automatically created
- Subsequent: Loads existing

### "No similar items found"
- Vector store is empty
- Create more plans/healings
- Try different search terms

### "HuggingFace download slow"
- First time downloads model (~90MB)
- Cached for future use
- One-time operation
